# Security Module for Claude Skills

Этот модуль предоставляет функции безопасности для всех скриптов в директории `skills/`:
- **Аудит-логирование** — запись всех важных событий в файл
- **Проверка целостности SKILL.md** — защита от модификации инструкций

## Компоненты

### `security.sh`
Общий модуль безопасности с функциями:
- `init_audit_log(script_name)` — инициализация аудит-логирования
- `audit_log(level, message)` — запись в аудит-лог
- `verify_skill_checksums(skills_dir)` — проверка целостности SKILL.md файлов
- `generate_skill_checksums(skills_dir)` — генерация checksums

### `generate-checksums.sh`
Утилита для генерации checksums всех SKILL.md файлов.

## Использование

### Первоначальная настройка

1. Сгенерируйте checksums для всех SKILL.md файлов:
```bash
cd skills/_common
./generate-checksums.sh
```

Или укажите путь к директории skills:
```bash
./generate-checksums.sh ~/.claude/skills
```

### Автоматическая проверка

Все скрипты автоматически:
- Инициализируют аудит-логирование при запуске
- Проверяют целостность SKILL.md файлов
- Логируют важные события

### Просмотр аудит-лога

Аудит-лог хранится в: `~/.claude/skills/audit.log`

Формат записей:
```
[2026-01-25T12:34:56Z] | ScriptName | LEVEL | Message
```

Уровни логирования:
- `START` — запуск скрипта
- `INFO` — информационные сообщения
- `WARN` — предупреждения
- `ERROR` — ошибки
- `SECURITY` — события безопасности

### Проверка целостности

При каждом запуске скрипты проверяют checksums SKILL.md файлов. Если файл был изменен, в лог записывается предупреждение уровня `SECURITY`.

## Файлы

- `security.sh` — модуль безопасности (используется всеми скриптами)
- `checksums.sha256` — база данных checksums (генерируется автоматически)
- `audit.log` — файл аудита (создается в `~/.claude/skills/audit.log`)

## Безопасность

Все файлы создаются с ограниченными правами доступа:
- Директории: `chmod 700`
- Файлы: `chmod 600`

Аудит-лог и checksums защищены от несанкционированного доступа.

## Обновление checksums

После изменения любого SKILL.md файла необходимо обновить checksums:

```bash
cd skills/_common
./generate-checksums.sh
```

Или из корня проекта:
```bash
./skills/_common/generate-checksums.sh
```

## Устранение неполадок

### Checksums не найдены
При первом запуске checksums могут отсутствовать — это нормально. Скрипты продолжат работу, но запишут предупреждение в лог.

### Ошибка доступа к аудит-логу
Убедитесь, что директория `~/.claude/skills/` существует и доступна для записи.

### Checksum verification failed
Если проверка целостности не прошла:
1. Убедитесь, что SKILL.md файлы не были изменены
2. Если изменения были намеренными, обновите checksums:
   ```bash
   ./skills/_common/generate-checksums.sh
   ```
