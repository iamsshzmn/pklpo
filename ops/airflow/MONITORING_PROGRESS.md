# Мониторинг прогресса выполнения DAG features_calc

## Как понять, что работа идёт

### 1. Логи в Airflow UI

После запуска команды (строка `⏰ Запуск команды...`) в логах будут появляться:

#### Начальная информация (сразу после запуска):
```
📊 План работы: N символов × M таймфреймов = K задач
📈 Символы: BTC-USDT-SWAP, ETH-USDT-SWAP, ...
⏰ Таймфреймы: 1m, 5m, 15m, ...
🎯 Индикаторов для расчёта: 176
🚀 Начинаем обработку...
```

#### Heartbeat-логи (каждые 30 секунд):
```
💓 ПРОГРЕСС: 15/120 задач (12.5%) | Баров: 45,000 | Индикаторов: 7,920,000 | Сохранено: 45,000 | Текущая задача: BTC-USDT-SWAP 1m | Скорость: 0.50 задач/сек | Осталось: ~3.5 мин
```

#### Итоговая статистика (в конце):
```
📊 ИТОГИ ЭТАПА FEATURES:
⏰ Время выполнения: 245.3 сек
📈 Обработано символов: 10
📊 Обработано баров: 120,000
🎯 Рассчитано индикаторов: 21,120,000
💾 Сохранено записей: 120,000
⚡ Скорость: 489.2 баров/сек
```

### 2. Файл логов внутри контейнера

Логи также пишутся в файл `/tmp/pklpo/features.log` внутри контейнера Airflow.

Проверка из хоста:
```bash
docker exec -it <airflow-scheduler-container> tail -f /tmp/pklpo/features.log
```

### 3. Мониторинг через базу данных

Проверка количества записей в таблице `indicators`:

```sql
-- Общее количество записей
SELECT COUNT(*) FROM indicators;

-- Записи за последний час
SELECT COUNT(*)
FROM indicators
WHERE updated_at > NOW() - INTERVAL '1 hour';

-- Прогресс по символам
SELECT symbol, timeframe, COUNT(*) as records
FROM indicators
WHERE updated_at > NOW() - INTERVAL '1 hour'
GROUP BY symbol, timeframe
ORDER BY records DESC;
```

### 4. Признаки того, что работа идёт

✅ **Работа идёт, если:**
- В логах появляются heartbeat-сообщения `💓 ПРОГРЕСС` каждые 30 секунд
- Количество записей в БД увеличивается
- В логах видны сообщения вида `✅ BTC-USDT-SWAP 1m: 1000 баров, 176 индикаторов, 1000 сохранено`
- Статус задачи в Airflow UI остаётся `running` (не `failed`)

❌ **Работа не идёт, если:**
- Логи обрываются на `⏰ Запуск команды...` и больше ничего не появляется
- Нет heartbeat-сообщений более 1 минуты
- Статус задачи `failed` или `up_for_retry`
- Количество записей в БД не увеличивается

### 5. Что делать, если прогресс не виден

1. **Проверить, что команда запустилась:**
   - В логах должна быть строка `🚀 ЗАПУСК ЭТАПА FEATURES`
   - Должна быть строка `📊 План работы: ...`

2. **Проверить файл логов:**
   ```bash
   docker exec -it <airflow-scheduler-container> cat /tmp/pklpo/features.log | tail -50
   ```

3. **Проверить процесс:**
   ```bash
   docker exec -it <airflow-scheduler-container> ps aux | grep "src.cli.main features"
   ```

4. **Проверить использование ресурсов:**
   ```bash
   docker stats <airflow-scheduler-container>
   ```

### 6. Ожидаемое время выполнения

- **1 символ × 10 таймфреймов:** ~2-5 минут
- **10 символов × 10 таймфреймов:** ~20-50 минут
- **Все символы (50+) × 10 таймфреймов:** 1-3 часа (зависит от объёма данных)

**Рекомендация:** Для тестирования используйте `--limit 1000` или конкретные `--symbols` для быстрой проверки.
