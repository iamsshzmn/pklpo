

## ✅ **1. Надёжность и отказоустойчивость**

* [ ] Все блоки `try/except` логируют ошибки с `exc_info=True`
* [ ] Повторные попытки при сбоях (retries, retry_delay) настроены в Airflow
* [ ] Используется `ON CONFLICT DO UPDATE` или проверка существования перед вставкой
* [ ] Есть валидация входных данных (колонки, пропуски, дубликаты)
* [ ] Контроль NaN/Inf и длины результата после расчёта
* [ ] Все INSERT-операции обернуты в транзакции
* [ ] Установлен `execution_timeout` в задачах Airflow
* [ ] Реализованы unit-тесты на каждый индикатор
* [ ] Интеграционные тесты на запись в БД

## ✅ **2. Масштабируемость и расширяемость**

* [ ] Индикаторы реализованы как отдельные классы/функции с общим интерфейсом
* [ ] Есть централизованный реестр индикаторов (dict, registry, config)
* [ ] Индикаторы векторизованы (без Python-циклов)
* [ ] Поддерживается batch-обработка по тикерам или временным окнам
* [ ] Возможна параллельная обработка (multiprocessing или Airflow dynamic tasks)
* [ ] Модуль работает на float32 при необходимости (экономия RAM)
* [ ] Все зависимости (репозитории, конфиги) внедряются через аргументы

## ✅ **3. Мониторинг и отладка**

* [ ] Логирование в формате: timestamp, level, module, message
* [ ] Логи включают: run_id, symbol, дата, имя индикатора
* [ ] Есть сбор метрик: время расчёта, число строк, пиковая RAM
* [ ] Метрики выводятся в лог или в Prometheus Pushgateway
* [ ] Все ошибки логируются с traceback
* [ ] Настроен режим DEBUG с расширенным логом
* [ ] Логируются все SELECT и INSERT операции с таймингом

## ✅ **4. Чистота архитектуры и тестируемость**

* [ ] Отдельные модули для: расчёта, БД, оркестрации
* [ ] Соблюдается SRP: каждый класс делает одну задачу
* [ ] Используются абстракции (interface/protocol) для индикаторов и хранилищ
* [ ] Бизнес-логика не зависит от Airflow и БД
* [ ] Есть покрытие unit-тестами всех расчётных модулей
* [ ] Нет жёстких зависимостей, внедрение через аргументы или фабрики
* [ ] Используется линтер (`flake8`) и mypy (опционально)

* [ ] Используется psycopg2 или SQLAlchemy с параметрами
* [ ] Один connection на всю задачу
* [ ] Вставка результатов в режиме upsert / batch insert
* [ ] Индексы на (symbol, timestamp) в таблице
