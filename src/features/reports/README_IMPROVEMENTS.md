# Рекомендации по восстановлению расчёта и сохранения индикаторов

## Обзор применённых улучшений

Данный документ описывает реализацию рекомендаций по восстановлению расчёта и сохранения индикаторов в модуле `features/`.

## 1. Проблемы, которые были решены

### 1.1 Проблемы с DataFrame обработкой
- **Проблема**: Сложная логика слияния индикаторов приводила к потере данных
- **Решение**: Упрощена логика слияния в `core.py`, добавлена валидация данных

### 1.2 Проблемы с сохранением в базу
- **Проблема**: Несоответствие ключей (`ts` vs `timestamp`), неправильные UPSERT операции
- **Решение**: Унифицированы ключи, улучшены UPSERT операции в `infrastructure/database.py`

### 1.3 Проблемы с типами данных
- **Проблема**: Некорректная конвертация timestamp, невалидные значения
- **Решение**: Добавлена строгая валидация типов данных

## 2. Реализованные улучшения

### 2.1 Разделение расчёта и сохранения

**Новые модули:**
- `calc.py` - расчёт индикаторов и сохранение в parquet
- `save.py` - загрузка из parquet и сохранение в базу

**Преимущества:**
- Избежание ограничений XCom (>48 KB)
- Возможность отладки каждого этапа отдельно
- Повторное использование результатов расчёта

### 2.2 Улучшенная валидация данных

**Новый модуль:** `validation.py`

**Функции:**
- Валидация OHLCV данных перед расчётом
- Валидация рассчитанных индикаторов
- Валидация данных перед сохранением в базу
- Проверка качества данных

### 2.3 Улучшенная обработка ошибок

**Новый модуль:** `error_handling.py`

**Функции:**
- Комплексная обработка ошибок
- Retry механизмы с экспоненциальным backoff
- Детальное логирование ошибок
- Стратегии восстановления

### 2.4 CLI-хелперы для тестирования

**Новый модуль:** `cli.py`

**Команды:**
```bash
# Расчёт индикаторов
python -m features calculate input.csv output.parquet --symbol BTCUSDT --timeframe 1H

# Валидация данных
python -m features validate data.csv --data-type ohlcv

# Сохранение в базу
python -m features save output.parquet --symbol BTCUSDT --timeframe 1H

# Полный пайплайн
python -m features pipeline input.csv output.parquet --symbol BTCUSDT --timeframe 1H
```

```
├── cli.py              # CLI для тестирования
├── infrastructure/

└── core.py            # Улучшенная логика расчёта
```

## 4. Ключевые принципы

### 4.1 Разделение ответственности
- **Расчёт**: `calc.py` - только вычисления, сохранение в parquet
- **Сохранение**: `save.py` - только загрузка из parquet и сохранение в БД
- **Валидация**: `validation.py` - проверка качества данных
- **Обработка ошибок**: `error_handling.py` - комплексная обработка ошибок

### 4.2 Надёжность
- Валидация на каждом этапе
- Retry механизмы для временных ошибок
- Детальное логирование для отладки
- Graceful degradation при ошибках

### 4.3 Тестируемость
- CLI-хелперы для локального тестирования
- Возможность тестирования каждого этапа отдельно
- Валидация данных на каждом этапе

## 5. Использование

### 5.1 Локальное тестирование

```bash
# 1. Расчёт индикаторов
python -m features calculate data.csv indicators.parquet --symbol BTCUSDT --timeframe 1H

# 2. Валидация результатов
python -m features validate indicators.parquet --data-type features

# 3. Сохранение в базу
python -m features save indicators.parquet --symbol BTCUSDT --timeframe 1H

# 4. Полный пайплайн
python -m features pipeline data.csv indicators.parquet --symbol BTCUSDT --timeframe 1H
```

### 5.2 Интеграция в Airflow DAG

```python
from features.calc import compute_and_dump_parquet
from features.save import save_parquet_to_pg

# Task 1: Расчёт
def calculate_indicators(**context):
    result = compute_and_dump_parquet(
        df_ohlcv=df,
        symbol=symbol,
        timeframe=timeframe,
        output_path="/tmp/indicators.parquet"
    )
    return result['output_path']

# Task 2: Сохранение
def save_indicators(**context):
    parquet_path = context['task_instance'].xcom_pull(task_ids='calculate')
    result = save_parquet_to_pg(
        session=session,
        parquet_path=parquet_path,
        symbol=symbol,
        timeframe=timeframe
    )
    return result
```
