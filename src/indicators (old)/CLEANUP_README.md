# Очистка старых данных в Indicators

## 📋 Описание

Добавлена автоматическая очистка старых данных из таблицы `indicators` для поддержания актуальности данных и оптимизации производительности.

## 🔧 Функциональность

### Автоматическая очистка
- **По умолчанию**: При каждом запуске расчета индикаторов автоматически удаляются данные старше 24 часов
- **Настраиваемый период**: Можно изменить период очистки через параметр `cleanup_hours`

### Ручная очистка
- Возможность запуска очистки отдельно от расчета индикаторов
- Настройка периода очистки

## 🚀 Использование

### Через main_with_options.py
```bash
# Автоматическая очистка (по умолчанию)
python src/main_with_options.py --indicators

# Без очистки
python src/main_with_options.py --indicators --no-cleanup
```

### Прямой запуск calc_indicators.py
```bash
# С очисткой (по умолчанию)
python src/indicators/calc_indicators.py --parallel

# Без очистки
python src/indicators/calc_indicators.py --parallel --no-cleanup

# Настройка периода очистки (например, 48 часов)
python src/indicators/calc_indicators.py --parallel --cleanup-hours 48
```

### Программный вызов
```python
from src.indicators.calc_indicators import cleanup_old_indicators, main_parallel

# Только очистка
deleted_count = await cleanup_old_indicators(hours_old=24)

# Расчет с очисткой
await main_parallel(symbol=None, cleanup_old=True, cleanup_hours=24)

# Расчет без очистки
await main_parallel(symbol=None, cleanup_old=False)
```

## 📊 Параметры

### cleanup_old_indicators()
- `hours_old` (int): Удалять данные старше указанного количества часов (по умолчанию 24)

### main_parallel()
- `symbol` (str, optional): Конкретный символ для обработки
- `cleanup_old` (bool): Очищать ли старые данные перед расчетом (по умолчанию True)
- `cleanup_hours` (int): Удалять данные старше указанного количества часов (по умолчанию 24)

```
📊 Удалено записей: 1,234,567

📊 Осталось записей: 923,262
```

## ⚠️ Важные замечания

1. **Безопасность**: Очистка выполняется только для данных старше указанного периода
2. **Производительность**: Удаление старых данных улучшает производительность запросов
3. **Актуальность**: Гарантирует использование только свежих данных для расчетов
4. **Откат**: Можно отключить очистку флагом `--no-cleanup`

1. **Автоматическое обслуживание**: Не требует ручного вмешательства
